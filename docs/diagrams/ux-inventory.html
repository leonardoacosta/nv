Dashboard /
good ▾

Central overview: stat cards (obligations, active sessions, five-byte health, cold starts), CC Session widget, active Nexus session card, recent activity feed, quick links.

UX Issues
  • Stat cards show "0" for everything when daemon is disconnected — no visual differentiation between "zero data" and "offline". Add a disconnected overlay or dim the cards.
  • 6 stat cards in one row on wide screens creates a wall of numbers. Group into 2 rows: operational (obligations, active, health) vs performance (cold starts, five-byte).
  • Auto-refresh button has no indicator of refresh interval (how often?).
Suggestions
  • Add a greeting/time-of-day banner — "Good morning, Leo" with today's briefing summary. Personalizes the first touchpoint.
  • Show last-updated timestamp next to auto-refresh to build trust in data freshness.
Animation Opportunities
  • Stat card number counters: Animate values from 0 to target on load using countUp easing (CSS @property number interpolation). ~200ms per card, staggered 50ms.
  • Section reveals: Each section (stats, session widget, activity) should fade-in-up with 100ms stagger between groups, not all at once.
  • Activity feed items: Slide-in-left with 30ms stagger per row when section enters viewport.
  • Refresh button: Spin the icon 360deg on click, then fade-pulse the updated content.
Briefing /briefing
broken ▾

AI-generated daily briefing with history navigation. Should be the "morning coffee" page.

UX Issues
  • P0: Client-side exception crashes the entire page. Full-viewport error with no recovery. Likely a rendering issue with briefing data parsing.
  • No error boundary — one component crash takes down the whole page.
Suggestions
  • Add an error boundary wrapping the briefing renderer so history/controls remain usable on crash.
  • Show a skeleton with typewriter placeholder while briefing generates — sets expectation of AI processing time.
Animation Opportunities
  • Typewriter reveal: When briefing text loads, reveal paragraphs with a subtle typewriter/fade effect — reinforces the "AI generating" metaphor.
  • History navigation: Slide-left/slide-right transitions when moving between dates.
Obligations /obligations
improve ▾

Task/obligation management with status filtering (Active/History), owner tabs, expandable obligation cards with full description and action buttons (Confirm Done, Reopen, Cancel).

UX Issues
  • Action button overload: Every obligation shows Cancel, Confirm Done, Reopen simultaneously. These are mutually exclusive states — show contextual actions only (e.g., active items show "Done" + "Cancel", not "Reopen").
  • Dense stat row: 5 stat cards + tabs + filter = heavy cognitive load at top. Consider collapsing stats into a single summary bar.
  • No urgency indicators: All obligations look the same. Items approaching deadline should have visual warning (amber glow, countdown).
  • Activity panel on right side competes for attention with the main list. Unclear if it's global or scoped to selected obligation.
Suggestions
  • Contextual actions: Show only relevant buttons per state. Use icon buttons with tooltips instead of full text to reduce visual weight.
  • Deadline proximity: Add a subtle amber-to-red gradient on obligation cards approaching due date.
  • Inline expand: Expand obligation details inline instead of always-visible — save vertical space.
Animation Opportunities
  • Status change: When marking done, card should scale down slightly + fade to green, then slide out after 600ms.
  • Tab switch: Crossfade content between Active/History tabs (opacity 150ms).
  • Expand/collapse: Smooth height transition (max-height + opacity, 200ms ease) for obligation detail expansion.
  • List reorder: When an item is completed and removed from active, remaining items should slide up smoothly (layout animation).
Approvals /approvals
improve ▾

Queue-based approval flow: left list of pending items, right detail panel with description + Approve/Dismiss actions.

UX Issues
  • No urgency triage: "Medium urgency" label exists but all items look identical. Use color coding — red for urgent, amber for medium, default for low.
  • Approve button is very large (full-width green) while Dismiss is ghosted — bias is good for approve-heavy workflows, but consider if dismiss needs equal weight.
  • Queue count badge is only visible in nav and inline header — the page title should reflect count too.
Suggestions
  • Keyboard shortcuts: A for approve, D for dismiss, arrow keys to navigate queue. Power user flow.
  • Batch actions: Select multiple, approve/dismiss all.
Animation Opportunities
  • Approve/dismiss: Card slides out left (approved) or right (dismissed) with a green/red flash, next item slides up into position.
  • Detail panel: Crossfade when selecting different queue items (opacity + subtle translateX 4px).
  • Queue empty: When last item is cleared, celebratory confetti-fade or a gentle "All clear" animation.
Diary /diary
good ▾

Daily interaction log with date navigation, stat cards (interactions, tokens, latency), and chronological event entries with tags.

UX Issues
  • Date navigation is minimal — just left/right arrows. No calendar picker or jump-to-date for reviewing past days.
  • Tag pills (tool names) are informational but not interactive — clicking a tag should filter the diary to that tool.
Animation Opportunities
  • Date transition: Slide diary entries left/right when navigating dates (like a carousel).
  • Entry stagger: Diary entries fade-in-up with 40ms stagger on load.
  • Stat counters: Same countUp animation as dashboard stat cards.
Sessions /sessions
good ▾

List of all Nexus agent sessions with search, project filter dropdown, and status tabs (All, Active, Idle, Completed).

Suggestions
  • Absorb CC Session page: Add a "CC Session" panel/tab here since both manage sessions. Reduces nav count.
  • Real-time indicators: Active sessions should pulse or have a subtle breathing animation to show they're live.
Animation Opportunities
  • Status tab switch: Crossfade content between filter tabs.
  • Session cards: Staggered fade-in-up (already present, verify stagger timing).
  • Active session pulse: Subtle border glow pulse (box-shadow animation, 2s infinite) on active session cards.
Messages /messages
improve ▾

Channel message history with search, channel filter tabs, and date range. Displays message log from Nexus conversations.

UX Issues
  • Dense text wall: Messages are displayed as uniform rows with no visual hierarchy. Long messages truncate without expand affordance.
  • Channel tabs are too subtle — small text filters with no visual differentiation. Consider pill-style tabs with channel color coding.
  • No message grouping by conversation or time window — it reads like a flat log, not a conversation view.
Suggestions
  • Time grouping: Group messages by hour or conversation thread with subtle dividers.
  • Expand long messages on click instead of truncating permanently.
  • Channel color coding: Assign each channel a subtle accent color for quick scanning.
Animation Opportunities
  • New message arrival: Slide-in-down from top with highlight flash (via WebSocket real-time feed).
  • Message expand: Smooth height reveal (max-height transition 200ms).
  • Channel switch: Crossfade between channel message lists.

Contact directory with search, category filters (All, Work, Personal, Contributor, Social), and "New contact" CTA.

Projects /projects
good ▾

Daemon-registered project listing with search and issue count. Uses accordion expansion for project details.

Suggestions
  • Project health indicators: Show a small status dot (green/amber/red) per project based on recent session success rate.
  • Quick stats: Show session count, last active time inline without needing to expand.
Animation Opportunities
  • Accordion expand: Smooth height + opacity transition (currently likely instant). 200ms ease-out.

Integrations /integrations
good ▾

Connected channels (Telegram), tools, and services (Agent, Alert_rules, Autonomy, Daemon, etc.) with status indicators and initials avatars.

UX Issues
  • Initials avatars (AG, AL, AU, DA) are functional but bland — all same dark gray. Use hashed colors from the service name for visual differentiation.
  • "Disconnected" dominates — when most services are disconnected, the page feels like an error state. Consider dimming disconnected items and elevating connected ones.
Animation Opportunities
  • Connected pulse: Subtle green glow pulse on "Connected" status badges (2s ease-in-out infinite).
  • Section stagger: Channels, Tools, Services groups fade-in-up with 100ms stagger between groups.
  • Status change: When a service connects/disconnects via WebSocket, animate the status badge transition (color crossfade + scale pop).
Usage /usage
improve ▾

Cost analytics with stat cards (cost subscriber, cost/time, total), token usage bars, and tool invocation table with usage bars.

UX Issues
  • $0.0000 everywhere when daemon is off — same "zero vs offline" problem as dashboard. Show "No data" or "--" instead of literal zeros.
  • Tool usage bars are very small and hard to read at a glance. Needs more vertical space or a proper bar chart.
  • No time range selector — is this today? This week? All time? Ambiguous.
Suggestions
  • Absorb Cold Starts as a tab here — both are performance/analytics views.
  • Time range picker: Today / 7d / 30d / All time toggle.
  • Sparkline charts for cost trends over time, not just point-in-time values.
Animation Opportunities
  • Bar chart reveal: Bars grow from left to right on load (width transition 400ms ease-out, staggered).
  • Cost counters: countUp animation for dollar values.

CC Session /session
improve ▾

Claude Code container session manager — Start/Stop/Restart controls, uptime/messages/last-activity stats, container log viewer.

UX Issues
  • Standalone page for one session — this is just a control panel card. Could be a panel within the Sessions page.
  • Log viewer shows "No log output yet..." even when stopped — should say "Session is stopped" instead.

Settings /settings
good ▾

Daemon configuration — 28 settings organized in a clean list with inline editing. Includes toggles, text inputs, and "restart required" warnings.

UX Issues
  • 28 settings in a flat list is a lot. Group into categories (General, Network, Scheduling, Advanced) with collapsible sections.
  • "restart required" label is amber text next to the field — easy to miss. Make it a more prominent banner when any restart-required field has been changed.
Suggestions
  • Dirty state indicator: Show unsaved changes count and a floating "Save & Restart" bar when restart-required fields are modified.
  • Group settings into expandable sections with counts.
Animation Opportunities
  • Save confirmation: Brief green flash on saved field (background-color transition 300ms).
  • Section collapse: Smooth height transition for grouped sections.
  • Toggle switch: Spring-physics feel on toggle interaction (slightly overshoot then settle).
Cross-Cutting Animation System

These animations should be implemented as reusable CSS utilities or a thin React wrapper. No animation library needed — CSS keyframes + transitions cover everything here.

Animation Technique Duration Where Used
fade-in-up Existing. opacity 0→1, translateY 8px→0 200ms ease Page content, cards, empty states
countUp CSS @property --num with transition: --num 800ms, or 15-line JS hook 600-800ms ease-out All StatCards, cost values, percentiles
slide-out-left translateX(0→-100%), opacity 1→0 300ms ease-in Approval approve, obligation done
slide-out-right translateX(0→100%), opacity 1→0 300ms ease-in Approval dismiss
crossfade opacity 1→0→1 on content swap 150ms ease Tab switches, panel transitions, file select
height-reveal grid-template-rows: 0fr→1fr + opacity 200ms ease-out Accordions, expand/collapse, detail panels
bar-grow width: 0→target% or scaleX(0→1) 400ms ease-out, staggered Usage bars, token bars, latency bars
glow-pulse box-shadow green/amber keyframe, infinite 2s ease-in-out Active sessions, connected integrations, running CC session
path-draw SVG stroke-dasharray/offset animation 800ms ease-out Cold Starts latency chart line
slide-date translateX(±20px) + opacity, direction-aware 200ms ease Diary date nav, briefing history nav
